# 热重载系统迁移方案

> 参照：`dof-frida-模块化开发模板`，基于其 mini-require 核心、Hook 注册表、模块管理器
> 
> 目标：将本项目从 `Script.load()` + `globalThis` 架构迁移为 `new Function` + `context` 架构，实现运行时热重载（`//reload`），无需 `docker restart`

---

## 一、核心改造：frida.js 重写

### 1.1 新增：`_require(path, context, name)`

替代 `Script.load()`。用 `new Function("module", "context", src)` 隔离执行模块代码。

```javascript
function _require(path, context, name = '') {
  if (_moduleCache[path]) return _moduleCache[path]
  const src = readFile(path, 'r', 10 * 1024 * 1024)
  const module = { name, exports: {} }
  new Function("module", "context", src)(module, context)
  _moduleCache[path] = module.exports
  return module.exports
}
```

**关键变化**：模块不再通过 `globalThis` 共享，而是通过 `context` 注入。

### 1.2 新增：`_hookManager`

集中管理所有 `Interceptor.attach/replace`，支持多模块共享同一地址、卸载时自动 detach/revert。

- `register(address, hookInfo, moduleName)` — Interceptor.attach/replace
- `unregister(address, moduleName)` — detach/revert，当无模块监听时清理

### 1.3 新增：`_moduleManager`

负责加载/卸载/重载单个或全部模块。

- `_loadModule(config)` — _require → init() → 注册 hooks
- `_unloadModule(name)` — dispose() → 注销 hooks → 清缓存
- `loadAll()` / `unloadAll()` — 批量操作
- `reload(moduleName)` — 热重载入口

### 1.4 新增：`context` 对象

替代 `globalThis` 作为模块间通信的唯一通道。

```
context
├── main: { log, LOG_LEVELS, readFile, loadConfig, api_mkdir, module: { reload } }
├── config: { common配置, db_config, drop_rate, ... }
├── nativeBindings: 所有 NativeFunction
├── databaseApi: MySQL 函数和句柄
├── gameApi: 通用 API
├── villageState: 怪物攻城常量
└── ...（基础模块自动挂载）
```

### 1.5 改造：`rpc.exports`

```javascript
rpc.exports = {
  init(stage, parameters) {
    if (stage === 'early') { awake() }  // Hook checkArgv → _start()
    else { _start() }                   // 热重载直接启动
  },
  dispose() { _dispose() }
}
```

### 1.6 保留不变

- `bootLog` / `LOG_LEVELS` / `console` 重写 — 已实现
- `frida.config` — Frida Gadget 配置
- `deploy.sh` — scp 路径不变

## 三、模块改造：21 个模块全部改格式

### 3.1 改造前（当前格式，以 dropRate.js 为例）

```javascript
// ============================================================================
// DNF Frida modern modular package - dropRate.js
// ...
// ============================================================================

let dropRateMultiplier = 2.0

function installDropRate() {
  Interceptor.replace(target, new NativeCallback(function(a1,a2,roll,a4) {
    roll = Math.floor(roll * dropRateMultiplier)
    return cLuckPointGetItemRarity(a1, a2, roll, a4)
  }, 'int', ['pointer', 'pointer', 'int', 'int']))
}

if (!globalThis.dnfPlugin) { globalThis.dnfPlugin = {} }
__dnfExport({ installDropRate, installDropRateHook })
```

### 3.2 改造后（新格式）

```javascript
/**
 * dropRate.js
 * 通过 CLuckPoint::GetItemRarity Hook 实现爆率倍率控制。
 */

var _config = { multiplier: 2.0 }

// 接口
const { log } = context.main
const { INFO } = context.main.LOG_LEVELS
const { cLuckPointGetItemRarity } = context.nativeBindings

// 变量
let dropCallCount = 0

// 导出
module.exports = {
  init() {
    _config.multiplier = context.config.drop_rate?.multiplier || 2.0
    log('爆率已启用, 倍率: ' + _config.multiplier)
  },
  dispose() {},
  hooks: [
    {
      address: '0x8550BE4',
      replace(a1, a2, roll, a4) {
        const newRoll = Math.min(Math.floor(roll * _config.multiplier), 1000000)
        return cLuckPointGetItemRarity(a1, a2, newRoll, a4)
      },
      retType: 'int',
      argTypes: ['pointer', 'pointer', 'int', 'int']
    }
  ]
}
```

### 3.3 改造规则对照表

| 旧 | 新 |
|----|----|
| `globalThis.xxx` | `context.xxx` |
| `bootLog(msg)` | `context.main.log(msg)` 或 `console.log` |
| `bootLog(msg, LOG_LEVELS.WARN)` | `console.warn(msg)` |
| `__dnfExport({...})` | `module.exports = { init, dispose, hooks }` |
| 手动 `Interceptor.attach(ptr, {...})` | `hooks: [{ address, onEnter, onLeave }]` |
| 手动 `Interceptor.replace + NativeCallback` | `hooks: [{ address, replace, retType, argTypes }]` |
| `globalConfig['xxx']` | `context.config.xxx` |
| `pluginFeatureSwitch.xxx` | 各模块 `_config` 或 `context.config.features.xxx` |
| `pluginSafeCall(name, cb)` | `_hookManager` 自动 try/catch |
| `__dnfMutable('x', getter, setter)` | 模块内部变量 + 挂到 context 上 |

### 3.4 各模块改造要点

| 模块 | 改造注意事项 |
|------|-------------|
| `config.js` | 不再作为模块，配置逻辑并入 frida.js 的 `_initMain()` |
| `coreRuntime.js` | 基础模块，pluginSafeCall 等挂到 `context.coreRuntime` |
| `nativeBindings.js` | 基础模块，所有 NativeFunction 挂到 `context.nativeBindings` |
| `databaseApi.js` | 基础模块，MySQL 函数挂到 `context.databaseApi` |
| `villageState.js` | 基础模块，常量挂到 `context.villageState` |
| `gameApi.js` | 基础模块，通用 API 挂到 `context.gameApi` |
| `packetApi.js` | 基础模块或合并入 gameApi |
| `main.js` | 删除，功能并入 frida.js 的 `_start()` |
| `legacyAliases.js` | 删除 |
| `gmCommands.js` | 改动最大：命令路由保留，Hook 声明化，`context` 引用 + `module.exports` |
| 其余 `features/*.js` | 模板化改造：`module.exports = { init, dispose, hooks }` |

## 五、风险与注意事项

| 风险 | 缓解措施 |
|------|---------|
| `new Function` 上下文不同 | 模板项目已验证，逐步迁移逐步验证 |
| `globalThis` → `context` 引用断裂 | 阶段 2 先确保基础 API 就位，再改功能模块 |
| `pluginFeatureSwitch` 丢失 | 保留在 `context.config.features` 或各模块 `_config` |
| `__dnfMutable` 状态管理丢失 | 改模块内变量，必要时挂到 context |
| `pluginSafeCall` 保护丢失 | 纳入 `_hookManager.register` 的 try/catch |
| 21 个模块改周期长 | 阶段 3 按简单→复杂排序 |
| `Interceptor.replace` 残留 | `_hookManager.unregister` 自动 revert |
